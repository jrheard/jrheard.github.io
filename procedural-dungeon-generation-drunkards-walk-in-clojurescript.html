Procedural Dungeon Generation: A Drunkard's Walk in ClojureScript

(def canvas-id (atom "canvas-1"))

(defn draw-grid
 [grid]
 (let [canvas (js/document.getElementById @canvas-id)
  ctx (.getContext canvas "2d")
  width (count (first grid))
  height (count grid)
  canvas-width (.-width canvas)
  canvas-height (.-height canvas)
  cell-width (/ canvas-width width)
  cell-height (/ canvas-height height)]

  (.clearRect ctx 0 0 canvas-width canvas-height)
  (set! (.-fillStyle ctx) "#CCC")

  (loop [x 0 y 0]

   (when (< y height)
	(when (= (-> grid
			  (get y)
			  (get x))
		   :empty)

	 (doto ctx
	  (.beginPath)
	  (.rect (* x cell-width) (* y cell-height) cell-width cell-height)
	  (.fill)))

	(recur (if (identical? (dec x) width) 0 (inc x))
	 (if (identical? (dec x) width) (inc y) y))))))

I‚Äôm working on a toy game, and figured it‚Äôd be fun to learn how to write code that generates random levels for it. I‚Äôd like to show you a simple algorithm for level generation called the Drunkard‚Äôs Walk. It generates levels that look like this:

As you can see, our level is a two-dimensional grid. Each cell on the grid is either empty or full. If it‚Äôs empty, the player can wander around in there and find monsters and gold and items and stuff. If it‚Äôs full, then it‚Äôs a cave wall and the player smacks into it.

The Drunkard‚Äôs Walk algorithm starts with a totally-filled-in level and then hollows it out one cell at a time, so let‚Äôs start by defining a function that creates a filled-in level.


(defn full-grid
 [width height]
 (vec (repeat height
	   (vec (repeat width :full)))))

Okay, so the Drunkard‚Äôs Walk algorithm looks like this:

  1. Pick a random cell on the grid as a starting point.
  2. If we‚Äôve carved out enough empty spots, we‚Äôre done.
  3. Walk one step in a random cardinal direction - north, south, east, or west, no diagonals - and carve out that new spot.
  4. Go back to step 2.

We‚Äôre almost ready to implement it, but first let‚Äôs define a little helper function that we‚Äôll use to make sure that we stay within the bounds of our grid and don‚Äôt try to walk off into the gaping void beyond.



(defn bound-between
 [number lower upper]
 (cond
  (< number lower) lower
  (> number upper) upper
  :else number))

(bound-between 10 0 9)

Okay, here we go!



(defn drunkards-walk
 [grid num-empty-cells]
 (let [height (count grid)
  width (count (first grid))]
  ; Guard against impossible demands.
  (when (<= num-empty-cells (* width height))

  (loop [grid grid
   ; Step 1: pick a random cell.
   x (rand-int width)
   y (rand-int height)
   empty-cells 0]

   ; Step 2: if we're done, return the grid.
   (if (= empty-cells num-empty-cells)
	grid

	(let [cell-was-full? (= (get-in grid [y x]) :full)
	 ; Step 3: walk one step in a random direction.
	 direction (rand-nth [:north :east :south :west])]

	 ; Step 4: back to step 2.
	 (recur (assoc-in grid [y x] :empty)
	  (case direction
	   :east (bound-between (inc x) 0 (dec width))
	   :west (bound-between (dec x) 0 (dec width))
	   x)
	  (case direction
	   :north (bound-between (dec y) 0 (dec height))
	   :south (bound-between (inc y) 0 (dec height))
	   y)
	  (if cell-was-full?
	   (inc empty-cells)
	   empty-cells))))))))

(-> (full-grid 40 40)
 (drunkards-walk 350)
 draw-grid)
